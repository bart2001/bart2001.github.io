L4 Mode

  • L4에는 2가지 모드가 존재한다.
  • Proxy 모드와 DSR(Direct Server Return) 모드로 나눌 수 있다.
    [출처] https://image.toast.com/aaaadh/alpha/2018/techblog/1%2817%29.png

Proxy Mode

  • Server에서 Client로 응답시 L4를 경유하여 응답하는 구조

  • Client → L4 → Server → L4 → Client

    장점

  • 같은 서브넷이 아닌 다른 IP대역이더라도 L4수용이 가능함

  • 하나의 포트에 여러가지 리얼 포트로 전달해 줄 수 있는 멀티 포트 기능 지원 가능함

    단점

  • 모든 트래픽이 L4를 경유하기 때문에 인터페이스 병목현상 및 장비에 부하를 줄 수 있음

  • Client IP를 전달하기 위해서는 L4에서 HTTP헤더 값을 추가하고 관련 가이드에 따른 AP 수정으로 지원 가능함 (HTTP/SSL프로토콜만 지원 가능함)

DSR Mode

  • 서버에서 클라이언트로 응답시 L4를 경유하지 않고 Direct 로 응답하는 구조

  • Client > L4 > Server > Client

    장점

  • L4를 경유하지 않고 Client로 바로 응답하여 L4장비 부하를 줄여 줌, 응답속도가 더 빠름

  • Client IP를 그대로 전달하여 서버에 로그를 남기거나 접근 제어를 할 수 있음

    단점

  • 같은 서브넷 안에 있는 서버(IP) 대역이 아닐 경우 L4수용이 어려움

  • 하나의 포트에서 여러가지의 리얼 포트로 전달해 줄 수 있는 멀티포트 기능지원이 어려움
